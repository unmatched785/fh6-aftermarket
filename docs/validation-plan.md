# 검증 GUI 시험 계획

## 목적

실제 반복 입력을 연결하기 전에 화면 판독과 사용자 제어가 환경 차이를 견디는지 확인한다.
검증 GUI는 이 기간 동안 게임 입력을 보내지 않는다. 오류가 생겨도 재시작하지 않으며,
사용자가 언제든 일시정지·재개·중지할 수 있어야 한다.

## 판정 원칙

- 같은 카드가 여러 프레임 유지되는 것은 한 번만 센다.
- 다른 카드로 갔다가 같은 차량명이 다시 나오면 `중복 관찰`로 기록하고 계속한다.
- 중복은 세 대 완료 수에 포함하지 않는다.
- OCR/캡처 실패는 세 번까지 재시도하고, 이후 `확인 필요`에서 기다린다.
- FH6가 전경이 아니면 판독만 대기한다.
- 목표 차량은 한 번의 신뢰 가능한 일치만으로도 `목표 차량 발견` 상태로 전환한다.
- 비목표 차량은 FH6 Meta 공식 차량 ID 후보로 정규화된 경우에만 완료 수에 포함한다.
- 공식 차량 후보를 만들지 못한 읽기 가능한 OCR도 `불확실`로 처리해 재시작 근거로 쓰지 않는다.
- 검증 모드에서는 목표가 없어도 게임을 재시작하지 않는다.

## 필수 환경 표

각 행에서 최소 5회씩 세 대 판독을 완료한다.

| 해상도/캡처 표면 | Windows 배율 | 창 모드 | 언어 | 결과 |
|---|---:|---|---|---|
| 1920×1080 | 100% | 테두리 없음 | KOR | 미검증 |
| 1920×1080 | 125% 또는 150% | 테두리 없음 | ENG | 미검증 |
| 2560×1440 | 100% | 테두리 없음 | KOR/ENG | 미검증 |
| 3840×2160 | 150% | 테두리 없음 | KOR | 아이콘·OCR·중복·물리 F2 실기 통과 |
| 사용자 PC 1 | 사용자 설정 | 테두리 없음 | KOR/ENG | 미검증 |
| 사용자 PC 2 | 사용자 설정 | 테두리 없음 | KOR/ENG | 미검증 |

독점 전체 화면은 별도 항목으로 시험한다. 캡처가 검게 나오거나 전경 판정이 불안정하면 첫
배포의 지원 모드는 테두리 없는 창 모드로 제한한다.

## 시나리오

### 정상 판독

1. 필터 적용 후 최대 확대된 지도에서 GUI를 시작한다.
2. 초록 아이콘 군집 감지 로그와 실제 캡처 크기를 확인한다.
3. 서로 다른 아이콘 세 개를 선택한다.
4. 세 차량명이 카드 첫 줄과 같고, 세 번째에서 `3대 검증 완료`가 되는지 확인한다.

### 중복/커서 오차

1. 차량 A를 판독한다.
2. 차량 B를 판독한다.
3. 차량 A를 다시 선택한다.
4. `중복 관찰`이 증가하지만 상태가 중지되지 않는지 확인한다.
5. 남은 차량 C를 선택해 정상 완료되는지 확인한다.

### OCR 불확실

1. 카드 가장자리에 커서를 두거나 전환 중인 프레임을 만든다.
2. 세 번까지 자동 재시도되는지 확인한다.
3. `확인 필요` 이후에는 더 진행하지 않는지 확인한다.
4. 커서를 바로잡고 `재개`하여 이어지는지 확인한다.

### 포커스와 긴급 중지

1. 판독 중 Alt+Tab으로 다른 창을 전경에 둔다.
2. 게임 입력 없이 `FH6 전경 대기`가 되는지 확인한다.
3. FH6가 전경인 상태에서 물리 F2를 누른다.
4. 0.5초 이내 GUI 세션이 중지되고 로그에 `F2 긴급 중지`가 남는지 확인한다.
5. 이후 실전 단계에서도 F2가 즉시 작동하는지 로그로 확인한다.

### 목표 차량

여섯 목표 차량의 제공 스크린샷으로 매처 회귀 검사를 유지한다. 라이브 목표 차량이 나오면
GUI가 정식명으로 `목표 차량 발견`을 표시하고 더 이상 판독을 진행하지 않는지 확인한다.

### 공식 차량명 정규화

1. `LaFerrari = |`가 `2013-ferrari-laferrari`로 정규화되고 목표 0건인지 확인한다.
2. `F8 Tributo = |`가 `2019-ferrari-f8-tributo`로 정규화되고 즉시 목표가 되는지 확인한다.
3. 선호 OCR이 LaFerrari여도 다른 OCR 모드가 F8 Tributo를 잡으면 목표가 우선하는지 확인한다.
4. `Aventador SVJ`가 LP700-4 목표로 오탐되지 않는지 확인한다.
5. DB 밖 문자열이 읽기 가능해도 `Uncertain`이고 3대 완료 수에 들어가지 않는지 확인한다.
6. 공식 DB에 여섯 목표 ID가 모두 존재하고 ID 중복이 없는지 확인한다.
7. `Ferrari 430 S$`가 `2007-ferrari-430-scuderia` 한 대로 복원되고 목표 0건인지 확인한다.

### Tesseract 설치 경로

1. ZIP의 `tessdata` 폴더에서 `eng.traineddata`와 `kor.traineddata`를 찾는지 확인한다.
2. 사용자 지정 `SCOOP` 루트와 일반 `Program Files\Tesseract-OCR` 설치를 각각 찾는지 확인한다.
3. `FH6_TESSERACT_EXE`, `FH6_TESSDATA_DIR` 직접 지정이 우선 적용되는지 확인한다.
4. 구성요소가 없으면 개발자 PC 경로 대신 확인한 후보와 해결 방법을 표시하는지 확인한다.

## 실전 입력 확대 원칙

전체 반복 입력용 `automationEnabled`는 계속 `false`로 둔다. 사용자가 승인한 오픈월드 첫
Enter만 별도 `practicalStartEnabled`로 허용하고, 다음 입력 묶음은 직전 실전 단계가 실제
화면에서 원하는 상태로 끝난 뒤에만 추가한다.

- 각 단계 진입 전에 정확한 FH6 전경과 16:9 캡처를 다시 확인
- 직전 단계의 화면 앵커가 없으면 다음 키를 보내지 않음
- 목표 발견·F2·포커스 이탈·인식 불확실이면 즉시 입력 중단
- 중복 차량명은 정지 사유가 아니라 커서 재시도 사유로 처리
- 다른 PC의 차이는 반복 횟수 할당이 아니라 실제 오류 로그로 보강

완전한 반복 루프를 연결할 때도 최대 횟수와 최대 실행 시간을 작게 제한한다.
