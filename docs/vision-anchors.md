# 화면 인식 앵커 계획

사용자가 제공한 세 장의 캡처는 모두 1920×1080입니다. 원본에는 게이머태그와 재화
정보가 있으므로 저장소에는 포함하지 않습니다.

## 1. 언어 목록

- 화면 특징: 청록색 설정 배경, 좌측 `언어 선택` 항목, 중앙 언어 목록
- 확인 대상: `한국어`, `English US`, 선택 테두리
- 기준 관심영역 후보: `(450, 220)`부터 `(1450, 920)`
- 언어가 KOR/ENG 중 하나로 확정되지 않으면 중단

## 2. 지도 필터

- 화면 특징: 중앙의 연두색 `Map Filter` 또는 한국어 대응 제목
- 확인 대상: `All`과 `Evolving World` 체크 상태
- 기준 관심영역 후보: `(615, 175)`부터 `(1310, 905)`
- `Evolving World` 단독 선택을 화면으로 확인한 뒤에만 필터 완료로 판단

차량 판독 단계에서는 별도의 `Aftermarket Cars` 필터를 사용합니다.

- 실화면 키 입력 기준으로 `Aftermarket Cars`까지 `Down ×26`
- 목록 순번만 계산하면 25칸이므로, 시작 포커스 차이를 고려해 문구 검출을 필수로 사용
- `All` 행에서 `Enter ×2`로 전체 항목을 해제한 뒤 `Down ×26`
- 행 문구와 체크박스를 모두 확인한 뒤에만 `Enter`로 활성화
- 필터가 다른 행에 있거나 `Aftermarket Cars` 체크가 확인되지 않으면 중단

## 3. 애프터마켓 지도 아이콘

- 화면 특징: 보라색 원형 배경과 흰색 지구본
- 제공된 FHD 캡처의 중심 후보: 약 `(355, 282)`
- 고정 좌표는 탐색 범위의 힌트로만 사용
- 실제 클릭 위치는 템플릿 검출 결과의 중심점
- 0개 또는 2개 이상 검출되면 중단

필터 직후, 지도 재진입 후, 선택 후를 별도 상태로 구분합니다.

- `filtered-map-translucent-marker`: 필터를 닫은 직후 반투명하게 표시된 보라색 지구본
- `map-closed-state`: 두 번째 `Esc`로 지도를 닫은 상태
- `aftermarket-map-marker-clear`: `Enter`로 지도를 다시 연 뒤 선명하게 표시된 지구본
- `aftermarket-location-card`: 아이콘에서 `Enter`를 누른 뒤 나타나는 정보 카드
- KOR 카드 제목: `이탈리안 엑조틱 대리점`
- ENG 카드 제목: `Italian Exotics Dealership`
- `fast-travel-confirmation`: `X`를 누른 뒤 나타나는 빠른 이동 확인창

반투명 아이콘 상태에서는 마우스 선택을 시작하지 않습니다. 선명한 아이콘과 정보 카드가
순서대로 확인되지 않으면 `X`를 누르지 않습니다.

### FHD 캡처 기준 초기 측정값

- 반투명 아이콘: 중심 `(353,285)`, 보라색 413픽셀, 평균 채도 0.616
- 선명 아이콘 영상 3프레임: 중심 약 `(354,284)`, 보라색 494~518픽셀, 평균 채도 0.654
- 정보 카드 선택 상태: 중심 `(352,286)`, 보라색 666픽셀, 카드 밝은 영역 29.26%
- 지도 필터 화면: 아이콘 후보 0개

이 값은 초기 관찰기 보정값이며, 실제 입력을 허용하기 전에 ENG/KOR 실게임 캡처를 더
수집해 경계값을 다시 검증합니다.

## 4. 지도상의 판매 차량 카드

판매 차량 아이콘은 애프터마켓에서 멀리 떨어져 있으면 지도에 나타나지 않습니다. 따라서
보라색 대리점 아이콘으로 빠른 이동해 가까이 도착한 뒤에만 전체 지도 검사를 시작합니다.
`Aftermarket Cars` 필터만 활성화해도 초기 배율에서는 판매 차량 아이콘이 보이지 않습니다.

- `exactly-three-aftermarket-car-icons`: 초록색 자동차와 `CR` 표식이 결합된 아이콘 정확히 3개
- 빠른 이동 뒤에는 차량을 움직이지 않아 플레이어 화살표와 판매 아이콘 군집이 고정 위치에서 겹침
- 위쪽 방향키를 유지하면 빠른 이동 때 남은 커서 위치를 중심으로 지도가 빠르게 최대 확대됨
- 최대 유지 시간은 임시로 1.5초이며 실제 실측 전까지 자동 실행하지 않음
- 아이콘이 3개보다 많거나 적은 상태에서는 카드 OCR을 시작하지 않음
- 플레이어 화살표를 피해 군집의 1시 방향을 먼저 호버해 `aftermarket-car-card-readable` 상태를 만듦
- 같은 중심을 두 번 읽지 않고, 서로 다른 차량 카드 3개가 모두 판독되어야 완료

2026-07-19 실측 캡처 표면은 `2560×1440`이었고 군집 경계는
`(956,662,94,103)`이었습니다. 과거 서로 다른 카드를 연 지점은 `(1005,690)`,
`(978,728)`, `(1006,707)`이었고, 새 기본값은 겹친 화살표를 피하는 1시 방향 호버부터
시작합니다. 구현에서는 경계 상자 상대 비율로 변환해 FHD/QHD/4K에
정규화합니다. 게임 내부 설정이 FHD여도 Windows 캡처 표면이 QHD일 수 있으므로 자동화는
항상 실제 캡처 크기를 기준으로 좌표를 계산합니다.

차량을 움직이지 않은 4K 실측에서는 군집 경계 `(1405,945,143,153)`에 대해
`(1508,1003)`에서 1번, `(1438,1043)`에서 2번 카드가 안정적으로 판독됐습니다.
`(1481,1012)`와 `(1492,1012)`는 모두 1번을 다시 판독해 세 번째 아이콘의 재현 가능한
지점이 아니었습니다. 사용자 표시를 같은 경계에 환산해 세 번째 기본점을 위쪽
`(약 1484,966)`, 상대 비율 `(0.55,0.14)`로 이동했습니다.

같은 4K 배치에서 플레이어 화살표 중심을 호버하면 `GoldCoyote5050 / Ferrari Dino`처럼
플레이어 카드가 열립니다. 이 화면은 판매 카드 검출기에서 `Uncertain / 미검출`로 처리되어
차량으로 오인되지 않았습니다. 세 번째 판매 아이콘은 화살표 중심이나 2번 높이에 머물지 않고,
겹친 군집 위로 노출된 작은 상단 영역에 진입합니다.

QHD 실화면에서 큰 정보 카드로 `Ferrari 812`, `LaFerrari` 두 이름을 확인했습니다.
차량명은 FH6 Meta 공식 차량 ID 후보로 정규화한 뒤 여섯 목표 ID와 대조합니다. 차량명 카드가
읽히지 않거나 공식 후보를 만들지 못하거나 목표 여부가 모호하면 결과는 `Uncertain`이며 언어
변경 재시작을 진행하지 않습니다.

## 정규화

관심영역과 검출 좌표는 게임 클라이언트 영역 기준으로 변환합니다.

```text
actualX = canonicalX × clientWidth / 1920
actualY = canonicalY × clientHeight / 1080
```

가로세로 비율이 16:9에서 허용 오차를 벗어나면 좌표를 사용하지 않습니다.

## 아직 필요한 화면

- English US 설정 화면에서 한국어를 선택하기 직전 상태
- 재시작 후 각 로딩 종료 시점
- 애프터마켓 차량 목록 전체 화면
- 목표 차량 등장 화면
- 목표가 아닌 정상 차량 화면 여러 장
