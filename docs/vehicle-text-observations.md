움직이지만 긴 파란 헤더 바로 아래의 첫 번째 영문 한 줄이 차량명이라는 구조는 동일했습니다.
지도 판독기는 파란 헤더를 먼저 찾고 해당 한 줄만 OCR합니다. 원본 실화면 캡처는 개인정보와
재화 정보가 있으므로 저장소에 커밋하지 않습니다.

`MC12 Corsa '08` 실화면의 검출 결과는 QHD 기준 헤더 `(720,217,559,49)`, 차량명 OCR
영역 `(736,271,520,64)`였습니다. Tesseract의 단일 행 모드가 `MC12 Corsa ‘08`을
신뢰도 96.7로 읽었고 최종 판정은 목표 없음(`Clear`)이었습니다.

2026-07-18에 제공된 여섯 장의 실제 등장 화면을 기준으로 작성했습니다. 원본 이미지는
다양한 해상도와 잘린 비율을 포함하고 개인정보도 표시하므로 Git에 넣지 않습니다.

## 비목표 텍스트 회귀 사례

같은 화면에 `Urus '19`, `Ferrari F12tdf`, `LaFerrari = |` 같은 비목표 차량도 함께 표시될
수 있습니다. 각각 공식 차량 후보로 정규화되며 여섯 목표 ID가 아니므로 목표로 판정되면
안 됩니다. `LaFerrari = |` 회귀 입력은 `2013-ferrari-laferrari` 한 대로 확정됩니다.

이전 3/3 회차에서 GUI가 그대로 표시했던 다음 OCR 문자열도 공식 비목표 차량 한 대씩으로
정규화합니다.

- `Huracan Tecnica` → `2022-lamborghini-huracan-tecnica`
- `Lambo Miura` → `1967-lamborghini-miura-p400`
- `Ferrari 430 S$` → `2007-ferrari-430-scuderia`

마지막 사례의 `$`는 잡음으로 제거하고 남은 `S`를 `Scuderia`의 첫 글자와 대조합니다. 세
결과 모두 공식 후보가 하나이고 여섯 목표 ID 밖이므로 비목표 판정입니다.

## 안전 판정

- OCR 모드 중 하나라도 목표 별칭 또는 목표 공식 ID 후보를 잡음: 즉시 정지
- 선호 OCR 결과가 공식 차량 후보를 하나도 만들지 못함: `UNCERTAIN`, 차량 수에 포함하지 않음
- 공식 후보가 하나 이상이고 후보 전체가 여섯 목표 ID 밖임: 비목표 차량으로 정규화
- 서로 다른 비목표 공식 ID 세 묶음이 확인됨: 이후 `CLEAR` 및 재시작 후보
- 일부 배너가 가려짐, OCR 누락, 화면 비율 불명: `UNCERTAIN`으로 정지

2026-07-19 17:44 KST에 `fh6meta.com/car-list`가 공개한 운영 목록은 627대입니다. 저장소의
`config/official-cars.json`은 이 공개 운영 데이터에서 생성하며, 릴리스 전 갱신 도구가 대수,
중복 ID, 여섯 목표 ID 존재 여부를 다시 검사합니다.

## 2026-07-18 OCR 검증 결과

Tesseract 5.5와 영어 언어 데이터를 사용해 제공된 여섯 장을 읽기 전용으로 검사했습니다.

- 전체 화면 1: 판매 배너 2개, `Aventador '12` 목표 검출, `Urus '19` 비목표 유지
- 전체 화면 2: 판매 배너 3개, `Lambo Sesto` 목표 검출, `Ferrari F12tdf` 비목표 유지
- 잘린 Diablo 화면 2장: 두 장 모두 `Diablo GTR` 목표 검출
- 잘린 F8 화면: `F8 Tributo '19` 목표 검출
- 잘린 Sesto 화면: `Lambo Sesto` 목표 검출

녹색 판매선의 길이·두께와 아이콘 근접성을 먼저 검사하고, 그 아래 모델명 영역만 4배
확대해 OCR합니다. 목표가 하나라도 확인되면 다른 배너의 판독 완전성과 관계없이 즉시
`TargetFound`입니다. 목표가 없을 때는 모든 검출 배너에서 읽을 수 있는 텍스트가 있어야만
하며 각 텍스트가 공식 차량 후보로 정규화되어야만 `Clear`가 됩니다. 배너가 없거나 하나라도
읽지 못하거나 DB에서 확정하지 못하면 `Uncertain`입니다.

이 정적 이미지 검사 시점에는 목표가 전혀 없는 실제 애프터마켓 전체 화면으로 `Clear`
판정을 검증하지 않았으므로 키·마우스 입력 자동화를 비활성 상태로 두었습니다.

## 2026-07-19 재시작 후 도착 화면 회귀 사례

English US 시작 화면부터 지도 필터, 애프터마켓 선택, 빠른 이동까지 키마다 화면을 확인해
도착했습니다. 온라인 플레이어 상태표시가 차량명 위를 가린 화면에서 기존 판독기는
`Petit PY (eae |)` 같은 잡음을 읽고도 `Clear`로 분류했습니다.

- 실제 화면: `Ferrari 812`, `LaFerrari` 등 비목표 차량
- 기존 결과: 배너 2개, 판독 2개, 목표 0개, 잘못된 `Clear`
- 수정: 최고 단어 신뢰도 70 미만의 문자열은 `Readable`로 인정하지 않음
- 추가 수정: `LaFerrari = |`를 공식 `2013-ferrari-laferrari`로 정규화하고 여섯 목표 ID와 대조
- 수정 후 기대 결과: 가려진 배너가 하나라도 있으면 `Uncertain`으로 정지

목표 별칭 매칭은 이 읽기 가능성 기준보다 먼저 적용되므로, 저화질 샘플에서 목표명이
잡힌 경우에는 기존처럼 즉시 `TargetFound`로 정지합니다.
